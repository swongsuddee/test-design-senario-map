# PP-171 · Permission Activation — Test Design

> Requirements → [PP-171_Permission_Activation.md](../requirements/PP-171_Permission_Activation/PP-171_Permission_Activation.md)
> Flow Diagram → [PP-171.diagram.md](./PP-171.diagram.md)
> Jira subtask → [PP-171](https://7-solutions.atlassian.net/browse/PP-171)

**Platform:** Web (Back Office — Admin Portal & Agency Portal)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Back Office (Admin BO + Agency BO)
**Scope:** Create Event access control by approval status, onboarding notification, corporate document gate, audit trail, RBAC enforcement, and Admin event list / approve-cancel flow

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Create Event — Access Control | State Transition | Three states (Pending / Rejected / Approved) determine locked/unlocked menu; exhaustive |
| Onboarding Notification | Specification-based | AC specifies push/email + onboarding link on Approve |
| Corporate Docs Gate | Decision Table | Docs incomplete (hidden) vs. complete (visible) — 2-row table |
| Audit Trail | Specification-based | AC specifies who + when must be logged |
| RBAC — Approve Permission | Error Guessing + Security | Non-KYC admin should receive 403; test boundary of role enforcement |
| Event List | EP | Paginated list + search/filter/sort — representative equivalence classes |
| Event Approve/Reject (Publish) | State Transition | pending_approved → published / draft |
| Event Approve/Reject (Cancel) | State Transition | pending_cancel → cancelled / published |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/web/admin/PP-171.permission-activation.web.ts`
- **State setup:** Use admin API to set Agency verificationStatus and Event status before each test
- **RBAC test:** Prefer direct API call with non-KYC auth token to ensure backend-level enforcement
- **Kafka side effects:** For TC-012 refund trigger, poll Payment Service API or consume Kafka events in test harness
- **Regression priority:** TC-001, TC-003, TC-007, TC-010, TC-012 as smoke suite; all 13 as full regression
