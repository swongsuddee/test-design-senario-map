---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Token Storage | Specification-based | AC 1.1 mandates Secure Storage; verify tokens are persisted and not stored in plain text |
| Auto Token Refresh | State Transition | States: valid token → expired (401) → refresh → new token → retry. Clear state transitions |
| Force Logout | State Transition + Error Guessing | Edge case: refresh token expired/invalid; system must force logout without hanging |
| Auth State Management | State Transition | Binary auth state (authenticated / unauthenticated) drives navigation outcomes |
| Navigation Guard | Decision Table | Two conditions: auth state × route protection level → 2 outcomes (allow / redirect) |

```
Notes:
  - TC-003 (silent refresh) and TC-004 (force logout) require ability to inject expired
    tokens into the app. This can be done via:
    (a) Network proxy (Charles / mitmproxy) to intercept and modify token values, or
    (b) A test-mode backdoor in the app to set token expiry.
  - TC-001 secure storage verification requires platform-specific inspection tools
    (iOS: idb / Keychain inspection; Android: adb shell + EncryptedSharedPreferences).
  - All 6 TCs are marked automation-candidate because the core behavior is testable
    at the API and state level, even if some setup requires test utilities.
```

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/auth/PP-20.auth-token-state.mobile.ts`
- **Automatable subset:**
  - TC-001: Partially automatable — login flow via Appium; secure storage inspection requires additional tooling (idb / adb).
  - TC-002: Fully automatable at API level using valid session token.
  - TC-003 & TC-004: Automatable with test setup hooks to inject expired tokens (via proxy or test backdoor).
  - TC-005 & TC-006: Fully automatable via Appium navigation assertions.
- **API-level testing:** Use STG API directly to verify token refresh endpoint behavior independent of UI.
- **Regression trigger:** Run before any release touching authentication, session management, or HTTP interceptor logic.
