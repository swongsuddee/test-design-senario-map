# PP-268 · [M-App][End-User] Event Detail (Mobile App) — Test Design

> Requirements → [PP-268_Event_Detail_Mobile.md](../requirements/PP-268_Event_Detail_Mobile/PP-268_Event_Detail_Mobile.md)
> Flow Diagram → [PP-268.diagram.md](./PP-268.diagram.md)
> Jira subtask → [PP-268](https://7-solutions.atlassian.net/browse/PP-268)

**Platform:** Mobile (iOS / Android)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Flutter App
**Scope:** End-user views event detail on mobile — renders event info fields, shows correct CTA (Join / Buy Ticket) based on event type and registration state, displays confirmed participant list

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Event Detail field rendering | Specification-based | AC 1.1 enumerates 6 required fields; verify each field is present and non-empty |
| CTA button type | Decision Table | Event type (Free / Paid) × Registration state (new / registered / sold out) → CTA variants |
| Participant list display | Specification-based | AC 2.1 requires only confirmed participants; AC 2.2 requires API integration |
| API error handling | Error Guessing | Network failure → error state; empty event → handle gracefully |
| CTA registration state | State Transition | States: enabled → registered → disabled (sold out); transitions driven by user action and data |

| API error simulation | STG proxy / mitmproxy to force 5xx on GET /events/{id} endpoint (TC-009) |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/event/PP-268.event-detail.mobile.ts`
- **Automatable subset:** TC-001 through TC-007, TC-010 through TC-012 — automatable via Appium + API pre-seeding
- **Manual / Partial:** TC-008 and TC-009 require data seeding at capacity or network mock
- **Platform note:** Run on both iOS and Android; flag any platform-specific rendering differences in TC-002
- **Regression trigger:** Run before every STG release touching the event detail API, mobile event detail screen, or participant list API
