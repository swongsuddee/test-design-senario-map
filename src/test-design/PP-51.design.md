# PP-51 · Create Event Running — Test Design

> Requirements → [PP-51_Create_Event_Running.md](../requirements/PP-51_Create_Event_Running/PP-51_Create_Event_Running.md)

**Platform:** Web (Backoffice)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Backoffice (BO) — Agency & Admin Portal
**Scope:** Create Event Running flow — กรอกข้อมูลงานวิ่ง, Ticket Types, Draft, Publish (pending_approved), Cancel Event บน BO

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Create Event Information | Specification-based + EP | ตรวจสอบการกรอกข้อมูลตาม AC แต่ละ field — valid/invalid partition |
| Ticket Types & Stock | Decision Table + BVA | หลาย field (ราคา, จำนวน, วันรับสมัคร) — boundary และ combination |
| Event Draft | State Transition | States: กำลังสร้าง → draft → resume → publish |
| Publish / Approval Flow | State Transition + Decision Table | Status: draft → pending_approved → published / draft (rejected) |
| Cancel Event | State Transition | Status: published → pending_cancel → cancelled / published (rejected) |
| RBAC / Access Control | Error Guessing | ทดสอบว่า non-Agency ไม่สามารถสร้าง Event ได้ |
