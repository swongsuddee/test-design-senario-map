## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Asset Preparation | Specification-based | AC1 specifies exact size (≥ 1024×1024 px, 1:1 ratio); verify asset meets spec before tooling runs |
| AppIcon — Multi-Flavor | Decision Table | 4 flavors × 2 platforms = 8 combinations; each flavor must display a distinct icon |
| Native Splash | Specification-based | AC3 specifies no-flicker, centered, correct background color; verify each attribute |
| Multi-Environment Build | Checklist / Specification-based | AC4 requires all 4 flavors verified on both Emulator and Physical Device |
| Negative — Wrong Icon / Stuck Splash | Error Guessing | Common integration failures: wrong flavor icon generated, splash screen stays stuck |

Notes:
  - All test cases are Manual (No) because:
    (a) App icon verification is a visual check on the device home screen — not accessible
        via Appium element selectors.
    (b) Native Splash Screen is rendered by the OS before Flutter engine starts — Appium
        cannot capture or assert its content.
  - Recommended approach: screenshot comparison using a visual regression tool (e.g.
    Percy, Applitools) integrated with device farm runs for scalable multi-flavor checks.
  - TC-008 explicitly requires physical devices to catch hardware-specific rendering bugs
    (e.g. notch handling, adaptive icon edge cropping on specific Android OEMs).

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/visuals/PP-35.app-icon-splash.mobile.ts`
- **Automation scope:** This is a visual/asset story. Direct UI automation via Appium is not applicable for icon and splash verification. Options:
  - CI pipeline script: run `dart run flutter_launcher_icons` for all flavors and assert exit code 0 + check generated file existence.
  - Visual regression: capture home screen screenshots via Appium after app install and diff against baseline images.
- **All 10 test cases are Manual** — requires QA to visually inspect icons on device home screen and splash during app launch.
- **Regression trigger:** Run before every release whenever `flutter_launcher_icons.yaml`, `flutter_native_splash.yaml`, or icon/splash assets are modified.
