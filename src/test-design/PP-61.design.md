**Platform:** Mobile (iOS / Android) + API

**Scope:** User Profile Epic — View/Edit Profile, Avatar Upload, Delete Account, Account Settings, Organizer Profile, Admin User History

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| View Profile (Public / Own) | Specification-based | AC กำหนดชัดเจน: public เข้าได้ทุกคน, own profile ต้อง authenticated |
| Edit Profile | EP + BVA | Valid/invalid partition สำหรับแต่ละ field + boundary length |
| Avatar Upload | Decision Table | 3 conditions: file type (JPG/PNG/other) × size (≤5MB / >5MB) |
| Delete Account | State Transition | States: active → soft_deleted → hard_deleted (30d grace) |
| Account Settings | Specification-based | Apply immediately — verify language change + notification pref |
| Organizer Profile | Decision Table | Verified (badge) vs Unverified (no badge) |
| Admin User History | EP + State Transition | Pagination + filter by action; RBAC guard |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/profile/PP-61.user-profile.mobile.ts`
- **GCS verification:** ต้องใช้ GCS client library แยกต่างหากหรือ API endpoint เพื่อ assert file resize
- **Hard delete (TC-011):** Automation ผ่าน admin API เพื่อ simulate grace period expiry; ไม่ใช่ UI
- **Kafka assertion:** ใช้ Kafka consumer test utility หรือ event log API ที่ backend จัดเตรียม
- **Regression trigger:** Run เมื่อมีการเปลี่ยนแปลง User Service, GCS pipeline, หรือ auth flow
