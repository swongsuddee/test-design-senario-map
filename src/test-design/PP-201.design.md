**Platform:** Mobile (iOS / Android)

**App:** POPPA Flutter App
**Scope:** End-to-end payment flow — Checkout, QR PromptPay, TrueMoney, Mobile Banking, Webhook idempotency, Refund policy, Organizer Payout

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Checkout — Order Summary | Specification-based | AC specifies price displayed clearly before confirm |
| Payment Method Selection | EP | Three valid methods in MVP1 (QR, TrueMoney, MobileBanking) + invalid selection |
| QR PromptPay — Generation | Specification-based | QR must be generated with 15-min TTL |
| QR PromptPay — Success | State Transition | PENDING → COMPLETED via webhook |
| QR PromptPay — Expiry | BVA + State Transition | TTL boundary: before 15 min (valid), at/after 15 min (expired) |
| Payment Status Polling | State Transition | PENDING → COMPLETED / FAILED / EXPIRED |
| TrueMoney Wallet | State Transition | Redirect → Return → Status resolved |
| Mobile Banking | State Transition | Redirect → Return → Status resolved |
| Webhook Idempotency | Error Guessing | Duplicate webhook must not double-write; first call processed, repeat ignored |
| Refund Policy | BVA + Decision Table | Three refund tiers: >7d (100%), 3–7d (50%), <3d (0%) — boundaries at 7d and 3d |
| Organizer Payout | Specification-based | Fee deduction + net transfer; accurate calculation |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/payment/PP-201.payment.mobile.ts`
- **Webhook simulation:** Use `axios.post(STG_BACKEND_WEBHOOK_URL, payload)` in test setup to simulate Payginix callbacks
- **QR Code assertion:** Assert QR image element is visible; extract paymentId from API response
- **Status polling:** Mock or observe `GET /payments/:id` response changes via API; assert UI reflects correct state
- **BVA dates:** Manipulate STG event end_date via API before test execution for refund boundary tests
- **Smoke suite:** TC-001, TC-002, TC-003, TC-005, TC-009
- **Regression suite:** All 14 TCs
- **Manual-only:** TC-007 (TrueMoney redirect), TC-008 (Mobile Banking redirect) — require physical device and real app-to-app redirect
