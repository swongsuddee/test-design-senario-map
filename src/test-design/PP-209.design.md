## A. Technique Selection

| Module | Technique | Rationale |

| Home Page — All Sections | Checklist / Specification-based | AC specifies 9 sections must all be present; enumerate each as a check |
| Event List — Interest-based | Decision Table | User has interests (Yes/No) × events shown (matching / default) — 2-row table |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/home/PP-209.home-main-page.mobile.ts`
- **Section presence:** Use Appium element selectors (accessibility IDs or Flutter widget keys) to assert each section container is visible on screen
- **Interest-based event list:** Intercept API response (via Appium network proxy or mock server) to assert interest query params; validate at least one event card category matches
- **Bottom Menu navigation:** Assert current page title / route after each tab tap using Flutter driver key or accessibility label
- **Visual regression:** Integrate visual snapshot test (Applitools / Percy) in CI pipeline for CI/Mood & Tone compliance; use baseline screenshots from Figma export
- **Smoke suite:** TC-001, TC-002, TC-005, TC-007
- **Regression suite:** All 7 TCs
