**Platform:** Web (Backoffice — Admin Portal)

**Scope:** Agency Verification Listing — แสดงตาราง, Search, Filter by Status, Review navigation, RBAC guard

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Dashboard List | Specification-based | AC กำหนด column list ชัดเจน — verify ทุก column ครบ |
| Search by Name | EP + Error Guessing | Partition: match / partial match / no match / empty input |
| Filter by Status | Decision Table | 4 status values (PENDING_REVIEW, APPROVED, REJECTED, REQUESTED_MORE_INFO) |
| Review Navigation | Specification-based | กด Review → navigate ไป PP-170 |
| RBAC Guard | Error Guessing | Non-Admin เข้า URL โดยตรง → ต้อง Redirect |
| Empty State | EP | API คืน empty list → ตารางแสดง empty state ถูกต้อง |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/web/admin/PP-104.agency-verification-listing.web.ts`
- **Platform:** Web BO — ใช้ WebdriverIO web driver (ไม่ใช่ Appium mobile)
- **API layer:** ใช้ GET /admin/organizers ผ่าน Admin JWT token สำหรับ assertion
- **Search debounce:** ถ้า search มี debounce delay ต้องใช้ `waitUntil` หลัง type
- **Filter assertion:** Assert จาก UI table rows และ API response status field
- **RBAC test:** ส่ง request ด้วย Agency JWT — expect 403 หรือ redirect
- **Regression trigger:** Run เมื่อมีการเปลี่ยนแปลง Admin Portal, BE List API, หรือ RBAC middleware
