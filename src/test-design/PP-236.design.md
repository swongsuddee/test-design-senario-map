# PP-236 · [BO][Organizer] Transaction & Payment (Finance Page) — Test Design

> Requirements → [PP-236_Transaction_Payment.md](../requirements/PP-236_Transaction_Payment/PP-236_Transaction_Payment.md)
> Flow Diagram → [PP-236.diagram.md](./PP-236.diagram.md)
> Jira subtask → [PP-236](https://7-solutions.atlassian.net/browse/PP-236)

**Platform:** Web (Back-Office / Organizer Portal)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Back-Office Web (Organizer Portal)
**Scope:** Organizer views finance list, verifies fee calculations (Gateway 2.5%, Platform 0.5%), and drills into Order Detail for a completed event

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Finance List Display | Specification-based | AC 1.1 mandates order/transaction list with statuses; AC 1.2 mandates specific financial summary fields |
| Fee Calculation | BVA / Specification-based | AC 3.1–3.3 specify exact percentages (2.5%, 0.5%) and formula; boundary values test rounding behaviour |
| Net Payout Formula | Decision Table | Three variables (Revenue, Gateway Fee, Platform Fee) → one outcome; test with known values |
| Order Detail fields | Specification-based | AC 2.2 mandates 6 specific fields: Order ID, Buyer, Ticket Type, Price, Status, Payment Date |
| Error / API failure | Error Guessing | Simulate API failure on finance list endpoint |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/web/organizer/PP-236.transaction-payment.web.ts`
- **Automatable subset:** TC-001 through TC-007, TC-009 through TC-012 — all automatable via WebdriverIO web UI + API pre-seeding
- **Manual / Partial:** TC-008 requires network mock; recommend WireMock stub or mitmproxy
- **Key assertion:** Fee calculations must be verified arithmetically — parse displayed values and compare to computed expected values within 0.01 THB tolerance
- **Regression trigger:** Run before every STG release touching the finance/payment API or organizer portal financial summary component
