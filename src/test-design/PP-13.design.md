## Test Data Requirements

| Data | Value / Source |
|------|----------------|
| Firebase projects | Dev, Staging, UAT, Prod Firebase project configurations |
| Android config files | google-services.json per flavor in `android/app/src/{flavor}/` |
| iOS config files | GoogleService-Info.plist per flavor in appropriate target directories |
| Flutter flavor build commands | `flutter run --flavor dev`, `--flavor staging`, `--flavor uat`, `--flavor prod` |
| Verification method | Firebase console → Project Settings → see active connections; or app startup log |
| Misconfiguration test | Remove google-services.json from one flavor directory before build |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/firebase/PP-13.firebase-integration.mobile.ts`
- **Automation scope:** This story is a configuration/infrastructure story. Direct UI automation is not applicable. Automation is limited to:
  - CI/CD build steps: `flutter build apk --flavor staging` / `flutter build ios --flavor staging` — assert exit code 0.
  - Log assertion: Parse app startup logs for Firebase project ID matching the expected flavor.
- **All 7 test cases are Manual** — requires developer/QA to inspect build output, Firebase console, and runtime logs.
- **Regression trigger:** Run before every release whenever Firebase config files, pubspec.yaml Firebase dependencies, or flavor configuration changes.
