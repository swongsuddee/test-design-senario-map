**Platform:** Web (Back Office — Agency Portal)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Back Office (Agency BO)
**Scope:** Book Bank account document upload field visibility and file validation for Individual and Corporate Agency profiles

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Upload Field Visibility | Decision Table | Two Agency types (Individual / Corporate) × field presence (Yes/No) — 2 conditions, exhaustive |
| File Upload — Valid Types | EP (Equivalence Partitioning) | Valid partition: PDF, JPG, PNG; Invalid partition: all other types |
| File Upload — Size Validation | BVA | Boundaries at 10 MB: valid ≤ 10 MB, invalid > 10 MB |
| Error Message Display | Specification-based | AC specifies error messages for unsupported type and oversized file |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/web/agency/PP-136.book-bank-upload.web.ts`
- **File injection:** Use `browser.execute` or `element.setValue(filePath)` for file input to bypass OS dialog
- **API validation:** After upload success, assert via `GET /agency/profile` that `bookBankDocumentUrl` is non-null
- **Priority:** Run TC-001 and TC-002 (visibility) as smoke on every STG deploy; TC-003–005 as regression
