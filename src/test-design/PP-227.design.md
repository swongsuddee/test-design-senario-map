**Platform:** Web (Backoffice — Organizer Portal)
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Backoffice (Organizer)
**Scope:** Forgot Password end-to-end flow — request link, token security, password reset validation, and post-success actions

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Email Format Validation | EP / BVA | Valid vs invalid email formats; boundary between accepted and rejected patterns |
| Rate Limit (60s cooldown) | BVA / State Transition | Boundary at exactly 60s; button state transitions (enabled → disabled → re-enabled) |
| Generic Security Response | Specification-based | AC 1.3 mandates identical response for found/not-found emails — fixed behaviour check |
| Token Security (expiry, one-time use) | State Transition | Token states: valid → expired, valid → used, new request → old invalidated |
| Password Strength Validation | EP / Decision Table | Matrix of missing character classes (uppercase, lowercase, digit, length) |
| Password Match Validation | Decision Table | Match vs mismatch × strength valid vs invalid |
| Password Reuse Guard | Error Guessing | New password = old password edge case |
| Show/Hide Password Toggle | Specification-based | Eye icon toggles input type between `password` and `text` |
| Post-Reset Actions | State Transition | Force logout → notification email → redirect + toast |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/auth/PP-227.forgot-password.mobile.ts`
- **Automatable subset (Yes):** TC-001–TC-006, TC-010–TC-013, TC-015 — all UI-driven form validation and happy-path flows.
- **Partial automation:** TC-007, TC-008, TC-009 require pre-seeded or time-manipulated tokens; TC-014 requires known current credentials; TC-016 requires multi-session verification.
- **Email inbox integration:** Consider Mailosaur or similar SMTP testing service to assert received emails in TC-001, TC-004, TC-009, TC-016.
- **Regression trigger:** Run before every STG release touching authentication, password policy, or email dispatch services.
