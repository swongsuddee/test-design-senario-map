**Platform:** Mobile (iOS / Android) + API
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Flutter App
**Scope:** Event Registration — Join Free Event, Buy Ticket (Paid), My Events, Leave Event, Check-in via QR Code

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Join Free Event | State Transition + EP | States: not_registered → registered; idempotency |
| Buy Ticket (Paid Event) | State Transition + BVA | States: reserved → confirmed / released; 15-min boundary |
| My Events | Specification-based | AC กำหนดข้อมูลที่ต้องแสดง: Upcoming/Past, QR Code |
| Leave Event | State Transition + Decision Table | Free vs Paid → different outcomes (refund or not) |
| Check-in via QR Code | State Transition + EP | Valid QR → checked-in; Invalid QR → error |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/event/PP-105.event-registration.mobile.ts`
- **Payment simulation (TC-003):** ใช้ Payginix STG environment; หรือ mock Kafka `payment.success` event โดยตรง
- **Timeout simulation (TC-005):** ใช้ admin API เพื่อ force-expire reservation; หรือ configure short timer บน STG
- **QR Code scan (TC-012):** ใช้ Check-in API โดยตรง (POST /checkin) แทนการ scan camera จริง สำหรับ automation
- **Dual-session:** TC-012, TC-013 ใช้ 2 Appium sessions (Organizer + Attendee) หรือ API + UI hybrid
- **Regression trigger:** Run เมื่อมีการเปลี่ยนแปลง Ticket Service, Payment integration, QR Code generation, หรือ Registration flow
