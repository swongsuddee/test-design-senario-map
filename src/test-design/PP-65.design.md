**Platform:** Mobile (iOS / Android) + API
**Framework:** WebdriverIO + Appium + Mocha (TypeScript)
**App:** POPPA Flutter App
**Scope:** Social Graph — Follow/Unfollow User, Followers/Following Lists, Block User (mutual), Kafka notification on follow

---

## A. Technique Selection

| Module | Technique | Rationale |
|---|---|---|
| Follow User | State Transition + EP | States: not_following → following; idempotency edge case |
| Notification on Follow | Specification-based | AC 1.2 กำหนด Kafka user.followed → notification |
| Unfollow User | State Transition | States: following → not_following; idempotency |
| Followers / Following Lists | EP + BVA | Pagination boundary; empty state |
| Block User | State Transition + Decision Table | Mutual block: A blocks B → 2 outcomes (A cannot see B, B cannot see A) |

## Automation Notes

- **Framework:** WebdriverIO + Appium + Mocha (TypeScript)
- **Spec file (future):** `src/test/mobile/social/PP-65.social-graph.mobile.ts`
- **Dual-session:** TC-009, TC-010 ต้องใช้ 2 sessions (User A + User B) พร้อมกัน
- **Kafka assertion:** ใช้ backend event log API หรือ Kafka consumer utility; ไม่ใช่ mobile-ui
- **Follow count assertion:** เรียก GET /profile API ก่อนและหลัง follow/unfollow เพื่อ compare count
- **Block visibility:** Assert ผ่าน API (GET /profile/{userId}) — ถ้า blocked ต้องได้ 403/404
- **Regression trigger:** Run เมื่อมีการเปลี่ยนแปลง User Service social graph, Notification Service, หรือ MongoDB schema
