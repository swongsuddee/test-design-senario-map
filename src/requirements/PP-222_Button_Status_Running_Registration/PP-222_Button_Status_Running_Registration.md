## User Story

> **ในฐานะ:** End-User ที่ต้องการสมัครวิ่งใน Event
> **ฉันต้องการ:** เห็นปุ่มที่มีชื่อและสีที่สื่อความหมายถูกต้องตามสถานการณ์ของการสมัคร
> **เพื่อ:** เข้าใจสถานะการสมัครของตัวเองได้ทันทีและดำเนินการได้ถูกต้อง

---

## 1. Description

จำแนกสถานะปุ่มสมัครวิ่ง (Button States & Naming) ในหน้า Event Running Registration โดยควรมีการเปลี่ยนชื่อปุ่ม (Label) และสี (Visual State) ตามสถานการณ์ที่แตกต่างกัน 5 สถานการณ์ เพื่อให้ผู้ใช้รับรู้สถานะได้ทันทีโดยไม่ต้องอ่านรายละเอียดเพิ่มเติม

---

## 2. Acceptance Criteria

### Scenario 1 — Coming Soon (ยังไม่เปิดรับสมัคร)

- **Given:** Event ยังไม่ถึงวันเปิดรับสมัคร
- **When:** ผู้ใช้เข้าชมหน้า Event
- **Then:** ระบบแสดงปุ่ม "รับสมัครเร็วๆนี้" และให้ผู้ใช้ลงทะเบียนรับการแจ้งเตือน (Notify Me) ได้

### Scenario 2 — First Time Registration (ยังไม่เคยสมัคร)

- **Given:** Event เปิดรับสมัครแล้ว และผู้ใช้ยังไม่เคยสมัครใน Event นี้
- **When:** ผู้ใช้เข้าชมหน้า Event
- **Then:** ระบบแสดงปุ่ม "เข้าร่วม" สีส้มเด่นชัด (Primary Action)

### Scenario 3 — Already Registered and Paid (สมัครและจ่ายเงินแล้ว)

- **Given:** ผู้ใช้สมัครและชำระเงินสำเร็จแล้วอย่างน้อย 1 ระยะ
- **When:** ผู้ใช้กลับมาที่หน้า Event
- **Then:** ระบบแสดงปุ่ม "สมัครอีก" สีรอง (Secondary) หรือมี Icon "+"

### Scenario 4 — Pending Payment (รอชำระเงิน)

- **Given:** ผู้ใช้สมัครแล้วแต่ยังไม่ได้ชำระเงิน
- **When:** ผู้ใช้เข้าชมหน้า Event
- **Then:** ระบบแสดงปุ่ม "ชำระเงิน" สีส้มเด่นชัด (Primary Action)

### Scenario 5 — Sold Out (เต็มทุกระยะ)

- **Given:** Event ปิดรับสมัครเนื่องจากที่นั่งเต็มทุกระยะ
- **When:** ผู้ใช้เข้าชมหน้า Event
- **Then:** ระบบแสดงปุ่ม "ปิดรับสมัคร" สีเทา (Disabled) ผู้ใช้กดไม่ได้

## 3. Technical Rules

| ลำดับ | สถานการณ์ (Scenario) | ชื่อปุ่ม (Button Label) | Action/State |
|-------|----------------------|-------------------------|--------------|
| 1 | ยังไม่เปิดรับสมัคร (Coming Soon) | รับสมัครเร็วๆนี้ | เปลี่ยนเป็นปุ่มลงทะเบียนรับการแจ้งเตือน (Notify Me) |
| 2 | ยังไม่เคยสมัคร (ครั้งแรก) | เข้าร่วม | ปุ่มสีส้มเด่นชัด (Primary Action) |
| 3 | สมัครให้ตนเองเสร็จแล้ว (จ่ายเงินแล้ว) | สมัครอีก | ปุ่มสีรอง (Secondary) หรือมี Icon "+" |
| 4 | สมัครแล้วแต่ "รอชำระเงิน" | ชำระเงิน | ปุ่มสีส้มเด่นชัด (Primary Action) |
| 5 | สมัครเต็มทุกระยะ (Sold Out) | ปิดรับสมัคร | ปุ่มสีเทา (Disabled) กดไม่ได้ |

---

## 4. Definition of Done

- [ ] ปุ่ม "รับสมัครเร็วๆนี้" แสดงเมื่อยังไม่เปิดรับสมัคร และ Notify Me ทำงานได้
- [ ] ปุ่ม "เข้าร่วม" สีส้มแสดงเมื่อยังไม่เคยสมัคร
- [ ] ปุ่ม "สมัครอีก" สีรองแสดงเมื่อสมัครและจ่ายเงินแล้ว
- [ ] ปุ่ม "ชำระเงิน" สีส้มแสดงเมื่อรอชำระเงิน
- [ ] ปุ่ม "ปิดรับสมัคร" สีเทา Disabled แสดงเมื่อ Sold Out
- [ ] ทดสอบการเปลี่ยนสถานะปุ่มครบทุก Scenario
